# Pantheora

Pantheora — where many agents become one intelligence.

Набор открытых сервисов для оркестрации ИИ‑агентов:

- **OpenClaw** как централизованный agent runtime (внутри — Pi и менеджер провайдеров LLM).[page:3][web:189]
- **Paperclip** как слой оркестрации «цифровых команд» и бизнес‑процессов.[page:0][page:1]
- **Forgejo** как самохостимый Git‑форж для кода, конфигураций и артефактов.[web:3][web:18]

Проект поставляется в виде `docker-compose`‑стека и минимального собственного кода — всё остальное опирается на существующие open‑source‑инструменты.

## Цели проекта

- Дать единый **операционный слой** поверх разных ИИ‑провайдеров и агентных фреймворков.
- Максимально использовать **готовые OSS‑компоненты** (OpenClaw, Paperclip, Forgejo, Pi) и свести «клей» к нескольким небольшим сервисам/адаптерам.
- Сделать стек, который можно:
  - запустить локально или на сервере одной командой `docker compose up`,
  - использовать как основу для построения «цифровой компании» без написания собственного оркестратора.

## Архитектура

### Общее представление

- **OpenClaw**

  - Централизованный agent runtime, внутри которого живёт Pi‑SDK (pi‑agent‑core, pi‑coding‑agent и т.д.).[page:3][web:193]
  - Управляет LLM‑провайдерами (Anthropic, OpenAI/Codex, Gemini, Ollama, OpenRouter, LiteLLM и др.).[page:2]
  - Поддерживает skills, MCP‑серверы, интеграции с Git, мессенджерами и сторонними сервисами.[web:64][web:5]

- **Paperclip**

  - Оркестратор цифровых команд: компании, роли, задачи, heartbeat‑циклы и бюджеты.[web:10][web:22]
  - Поддерживает adapters, включая **process‑adapter**, который запускает внешние рантаймы как процессы и общается через env/API.[page:1]
  - Для интеграции с OpenClaw будет использоваться кастомный adapter.

- **Forgejo**

  - Самохостимый Git‑форж (форк Gitea) с репозиториями, issues, PR, релизами и контейнерным registry.[web:3][web:18]
  - Является «источником правды» для кода, конфигураций и GitOps‑файлов.

- **Кастомный сервис** `paperclip-openclaw-adapter`
  - Лёгкий HTTP‑/process‑bridge между Paperclip и OpenClaw.
  - Преобразует задачи Paperclip в вызовы OpenClaw (определённый skill/agent) и возвращает результат.

## Сервисы в docker‑compose

Планируемые сервисы:

- `forgejo` — Git‑форж.
- `openclaw-gateway` — OpenClaw runtime (Pi + менеджер провайдеров).
- `paperclip` — Paperclip сервер + UI.
- `paperclip-openclaw-adapter` — прокси Paperclip → OpenClaw.
- (опционально) `opencode` — внешний coding‑агент для разработчиков, который можно использовать параллельно с Pi.[web:144][web:170]

Каждый сервис конфигурируется через `.env`/`env_file` и имеет отдельный volume для состояния.

## Поток данных (high‑level)

1. Пользователь/внешняя система создаёт задачу или бизнес‑цель в Paperclip.
2. Paperclip назначает её на цифровую команду (набор агентов) и через adapter отправляет запрос в OpenClaw.
3. OpenClaw, используя встроенный Pi‑runtime и настроенных LLM‑провайдеров, выполняет задачу (код, анализ, интеграции и т.п.).[page:3][page:2]
4. При необходимости OpenClaw взаимодействует с Forgejo (чтение/запись репозиториев, создание PR).
5. Результат возвращается в Paperclip, где отражается в UI (статус, артефакты, ссылки на PR и др.).

## Конфигурация провайдеров LLM

### Принципы

- **Единоразовая настройка** ключей и endpoint’ов — на уровне OpenClaw (auth‑профили, registry провайдеров).[page:2][web:211]
- **Гибкая настройка** моделей/режимов — на уровне:
  - конфигов OpenClaw (per‑agent, per‑workspace),
  - конфигов агента в Paperclip (какого провайдера/модель использовать для конкретной роли).[web:81][web:84]

### Источник правды

- Все API‑ключи и базовые URL провайдеров задаются через `.env` и конфиги OpenClaw:
  - прямые API (Anthropic, OpenAI, Gemini и др.),
  - gateway‑решения (LiteLLM, Cloudflare AI Gateway, OpenRouter, Vercel AI Gateway),
  - локальные модели (Ollama, vLLM/SGLang).[page:2]

## Быстрый старт (черновой план)

> ⚠️ Команды и имена сервисов пока условные и будут уточняться по мере разработки.

1. Склонировать репозиторий:

   ```bash
   git clone https://github.com/Web-Usov/pantheora-orchestrator.git
   cd pantheora-orchestrator
   ```

2. Создать `.env` из шаблона:

   ```bash
   cp .env.example .env
   # отредактировать ключи провайдеров, базовые URL и домены
   ```

3. Поднять стек:

   ```bash
   docker compose up -d
   ```

4. Пройти onboarding OpenClaw (auth провайдеров, модели).
5. Зайти в Paperclip, создать компанию/проект и подключить adapter к OpenClaw.

Подробные шаги будут описаны позже в разделе `docs/`.
