# Инфраструктура (Docker Compose)

Стек: **PostgreSQL 18**, **Redis 8**, **Forgejo 14**. Сеть Docker с фиксированным именем **`infra`**.

## Запуск и остановка

```bash
cd infra
docker compose up -d
docker compose ps
docker compose logs -f
docker compose down
```

Первый запуск Postgres выполняет скрипты из `./init/` (создание пользователя и БД для Forgejo). Повторно создать роль/БД для Forgejo вручную:

```bash
docker compose exec postgres sh /docker-entrypoint-initdb.d/01-forgejo.sh
```

Для произвольных приложений удобнее скрипты из **`./scripts/`** (см. ниже).

## Порты на хосте

| Сервис   | Порт на хосте | Внутри контейнера / сети |
|----------|---------------|---------------------------|
| Postgres | 5432          | `postgres:5432`         |
| Redis    | 6379          | `redis:6379`            |
| Forgejo HTTP | 3000    | `forgejo:3000`          |
| Forgejo SSH (git) | 222 | `forgejo:222`           |

Git по SSH везде использует порт **222**: и на хосте, и внутри контейнера (проброс `222:222`, `SSH_LISTEN_PORT` / `SSH_PORT` в Forgejo).

## Переменные окружения

Файл **`.env`** в каталоге `infra` (рядом с `docker-compose.yml`). Docker Compose подставляет значения из него автоматически. Шаблон без секретов для копирования: **`.env.example`** (`cp .env.example .env`, затем отредактируйте пароли).

| Переменная | Назначение |
|------------|------------|
| `POSTGRES_PASSWORD` | Пароль суперпользователя БД **`usov`** (в шаблоне указан `changeme` — смените) |
| `FORGEJO_DB_PASSWORD` | Пароль роли **`forgejo`** и подключения Forgejo к БД; можно оставить пустым — тогда используется `POSTGRES_PASSWORD` |

Файл **`.env`** добавлен в **`.gitignore`**, чтобы не попадал в репозиторий. В продакшене используйте надёжные пароли.

## Данные на диске

- `./data/postgres` — файлы кластера PostgreSQL  
- `./data/redis` — RDB/AOF Redis  
- `./data/forgejo` — данные Forgejo (`/data` в контейнере)

Полное пересоздание данных: остановить стек, удалить `./data`, снова `docker compose up -d`.

## Учётные записи PostgreSQL

- Суперпользователь: **`usov`**, БД по умолчанию с тем же именем — **`usov`**.  
- Отдельно для Forgejo: пользователь **`forgejo`**, БД **`forgejo`** (создаются init-скриптом).

## Скрипты: роли и базы (`scripts/`)

Нужны запущенный стек и файл **`.env`** с непустым **`POSTGRES_PASSWORD`**. Скрипты сами переходят в каталог `infra` и вызывают `docker compose exec` к контейнеру **`postgres`**, подключаясь суперпользователем **`usov`**.

Имена **`username`** и **`database`**: только латинские буквы, цифры и подчёркивание **`_`** (как безопасные идентификаторы в SQL).

### `scripts/create_user_and_db`

Создаёт роль PostgreSQL и базу с **одинаковым именем**.

```bash
./scripts/create_user_and_db <username> <password>
```

Пример:

```bash
cd infra
./scripts/create_user_and_db myapp 'надёжный_пароль'
```

Строка подключения (с хоста): `postgresql://myapp:надёжный_пароль@127.0.0.1:5432/myapp`.

Если роль или база с таким именем уже есть, скрипт завершится с ошибкой.

### `scripts/create_db`

Создаёт **новую базу** и назначает владельцем уже существующую роль **`username`**.

```bash
./scripts/create_db <username> <database>
```

Пример (роль `myapp` должна существовать):

```bash
cd infra
./scripts/create_db myapp myapp_reports
```

Если роли нет или база с именем **`database`** уже существует — ошибка.

## Как указывать URL и хосты для подключений

Правило: **откуда** идёт подключение определяет, **какое имя хоста и порт** нужны.

### 1. Другой контейнер в том же Compose (сеть `infra`)

Используйте **имя сервиса** и **внутренний порт** (как в таблице «внутри сети»):

- Postgres: `postgres`, порт `5432`  
- Redis: `redis`, порт `6379`  
- Forgejo по HTTP: `http://forgejo:3000`  
- Git по SSH к Forgejo с другого контейнера в сети `infra`: `ssh -p 222 git@forgejo` (или хост `forgejo`, порт `222` в URL клонирования).

Примеры строк подключения:

- `postgresql://forgejo:ПАРОЛЬ@postgres:5432/forgejo`  
- `redis://redis:6379`

`localhost` из такого контейнера указывает на **сам контейнер**, а не на хост.

### 2. Процесс на хосте (та же машина, вне Docker)

Опубликованные порты проброшены на **все интерфейсы** (`0.0.0.0`), поэтому с хоста обычно подходит:

- `127.0.0.1` или `localhost`  
- Postgres: `127.0.0.1:5432`  
- Redis: `127.0.0.1:6379`  
- Forgejo: `http://127.0.0.1:3000`  

Строки подключения:

- `postgresql://usov:ПАРОЛЬ@127.0.0.1:5432/usov` или `.../forgejo` с пользователем `forgejo`  
- `redis://127.0.0.1:6379`

### 3. Другая машина в локальной сети (LAN)

Нужен **IP хоста с Docker** в вашей сети (например `192.168.1.110`), а не `127.0.0.1`.

- Postgres: `192.168.1.110:5432`  
- Redis: `192.168.1.110:6379`  
- Forgejo: `http://192.168.1.110:3000`  
- Git по SSH: `ssh -p 222 git@192.168.1.110`

На хосте должны быть разрешены входящие соединения на эти порты (**firewall**, `ufw`, правила облака и т.д.). Публикация Postgres и Redis в LAN — **риск**: используйте сильные пароли, по возможности ограничьте доступ по IP или слушайте только нужный интерфейс (отдельная правка `ports` в compose).

### 4. Контейнер не из этого Compose, но на том же Docker

Варианты:

- Подключить контейнер к сети **`infra`**:  
  `docker network connect infra ИМЯ_КОНТЕЙНЕРА`  
  После этого доступны имена `postgres`, `redis`, `forgejo` как в п. 1.  
- Либо обращаться к сервисам через **хост**: `host.docker.internal` (часто на Docker Desktop) или IP шлюза Docker на Linux (часто `172.17.0.1`) и **опубликованные** порты (`5432`, `6379`, `3000`, `222`).

### Forgejo: публичный URL (ROOT_URL)

Если заходите в веб-интерфейс не с `localhost`, а по IP или домену, в настройках Forgejo (или через переменные окружения) нужно задать **корневой URL**, по которому пользователи реально открывают сайт, иначе ссылки, OAuth и клонирование могут вести на неверный хост.

Через compose можно добавить, например:

```yaml
FORGEJO__server__ROOT_URL: http://192.168.1.110:3000/
```

(замените на свой URL со слешем в конце, при HTTPS — `https://...`).

Порт SSH в мастере установки Forgejo и в настройках сайта должен быть **222** (совпадает с `FORGEJO__server__SSH_PORT` / `SSH_LISTEN_PORT` в compose). Если Forgejo уже был установлен раньше с портом **22**, после смены compose перезапустите контейнер и проверьте `app.ini` или админку «Конфигурация».
